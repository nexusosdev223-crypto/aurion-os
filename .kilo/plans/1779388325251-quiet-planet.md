# Plan: Start Aurion OS Autonomous Agent

## Objective
Start the AURI OS agent development environment by launching the Next.js dev server and the autonomous agent runtime.

## Current State
- Next.js 16.2.6 + React 19 project
- Available npm scripts: `dev`, `build`, `start`, `lint`
- Shell scripts: `agent-runtime.sh`, `run-executor.sh`, `run-fix-engine.sh`
- `.env.local` contains **placeholder Supabase credentials** — API endpoints `/api/metrics/velocity` and `/api/ledger` will return 500 errors until real keys are configured
- Ollama (`localhost:11434`) must be running for agent runtime; model `qwen2.5-coder:7b` is expected

## Steps

### Step 1 — Start the Next.js Dev Server
```bash
npm run dev
```
- Must run first — agent scripts depend on the dev server being live at `localhost:3000`
- Without valid Supabase credentials, API routes will return 500 with "Supabase query failed: Invalid API key"
- Dev server will be ready in ~300ms

### Step 2 — Start the Agent Runtime (separate terminal or background)
```bash
./agent-runtime.sh
```
- Fetches velocity metrics from `localhost:3000/api/metrics/velocity`
- Sends the prompt to Ollama at `localhost:11434` using `qwen2.5-coder:7b`
- Logs the agent decision in an infinite loop with 10-second intervals
- Requires Ollama to be installed and running with `qwen2.5-coder:7b` pulled

### Step 3 — (Optional) Run the Executor Engine
```bash
./run-executor.sh
```
- Runs `npx tsx src/app/api/metrics/velocity/executor.ts` directly
- Starts the autonomous autopilot executor

### Step 4 — (Optional) Run the Fix Engine Daemon
```bash
./run-fix-engine.sh
```
- Continuous background daemon that:
  - Ensures required directories exist
  - Auto-corrects stale relative imports to absolute path aliases
  - Monitors `/api/ledger` health every 4 seconds
  - Detects and warns on "Invalid API key" errors

## Caveats
- The Ollama dependency (`http://localhost:11434`) is external — make sure Ollama is installed and the `qwen2.5-coder:7b` model is pulled before running `agent-runtime.sh`
- Replace placeholder values in `.env.local` with real Supabase keys to get past the 500 errors on API routes — without them the agent will see `"Invalid API key"` errors every poll cycle

## Expected Outcome
- Page served at `http://localhost:3000`
- Terminal streaming velocity metrics and agent decisions every 10 seconds
- Fix engine warning about invalid Supabase keys until credentials are updated
